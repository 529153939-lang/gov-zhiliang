# 质量安全监督信息系统v1.1 产品需求文档

## 1. 产品概览
质量安全监督信息系统v1.1是一套面向政府农业农村局质量安全组人员的Web系统，用于监督和管理农业生产的质量安全。
- 系统旨在提供全面的农业生产主体管理、生产记录追踪、质量检测、溯源管理等功能，确保农产品质量安全。
- 系统通过集中化管理和可视化展示，提高监管效率，为农业质量安全决策提供数据支持。

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 角色权限 |
|------|---------------------|------------------|
| 系统管理员 | 预设账号 | 系统所有功能的访问和操作权限，包括系统管理功能 |
| 政府农业农村局质量安全组人员 | 系统管理员创建 | 系统所有功能的访问和操作权限，除系统管理功能外 |

### 2.2 功能模块
我们的质量安全监督信息系统包含以下主要页面：
1. **行政区划管理**：按丹东凤城城市区划逐级显示，包括区县、街道、办事处、乡镇、村、企业/农户、企业/农户对应的大棚地块资产明细。
2. **特色农产管理**：管理丹东特色农产的种类类型，如种植、腌制、水产等。
3. **经营主体管理**：管理涉农企业、专业合作社、家庭农场、包装中心、一般农户、经纪人等经营主体信息。
4. **生产管理**：管理生产记录明细，包括电子台账、生产记录查询、农事操作统计等。
5. **追溯管理**：管理溯源码生成和查询。
6. **检验检测管理**：管理检测信息。
7. **合格证管理**：管理承诺合格证。
8. **土地管理**：管理大棚地块信息、种植作物类型、规模面积、物联设备及土壤环境检测信息。
9. **购销台账**：管理购销台账监管信息。
10. **统计分析**：提供从业人员统计、产地环境监测看板、土地产值监测看板、农事操作统计、购销台账统计等可视化分析。
11. **气象监督**：管理气象监督信息。
12. **系统管理**：配置权限、账号设置等功能。

## 3. Core Process
### 3.1 主要用户操作流程
1. **登录流程**：用户通过系统登录页面输入用户名和密码，验证通过后进入系统主界面。
2. **行政区划管理流程**：用户进入行政区划管理页面，按照层级查看和管理行政区划信息。
3. **经营主体管理流程**：用户进入经营主体管理页面，查看经营主体列表，点击查看详情，管理经营主体信息。
4. **生产管理流程**：用户进入生产管理页面，查看生产记录明细，查看详情，管理生产记录。
5. **追溯管理流程**：用户进入追溯管理页面，管理溯源码生成和查询。
6. **检验检测管理流程**：用户进入检验检测管理页面，管理检测信息。
7. **合格证管理流程**：用户进入合格证管理页面，管理承诺合格证。
8. **土地管理流程**：用户进入土地管理页面，管理大棚地块信息、种植作物类型、规模面积、物联设备及土壤环境检测信息。
9. **购销台账流程**：用户进入购销台账页面，管理购销台账监管信息。
10. **统计分析流程**：用户进入统计分析页面，查看各类统计数据的可视化展示。
11. **气象监督流程**：用户进入气象监督页面，管理气象监督信息。
12. **系统管理流程**：用户进入系统管理页面，管理用户账号、角色和权限配置。

## 4. 用户接口设计
### 4.1 设计风格
- **主色**：38,98,183（RGB），相关衍生色为主
- **辅助色**：根据主色衍生的浅色和深色
- **按钮样式**：现代化圆角按钮，hover效果明显
- **字体**：系统默认无衬线字体，标题使用加粗样式
- **布局样式**：现代化紧凑型页面样式，整体白色底风格
- **图标样式**：使用简洁现代的线性图标

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI元素 |
|-----------|-------------|-------------|
| 登录页面 | 登录表单 | 用户名输入框、密码输入框、登录按钮、忘记密码链接 |
| 主界面 | 左侧菜单 | 一级菜单和二级菜单，展开/折叠功能 |
| 主界面 | 顶部导航 | 系统名称、用户信息、退出按钮 |
| 主界面 | 内容区域 | 面包屑导航、功能按钮、数据表格、分页控件 |
| 行政区划管理 | 行政区划列表 | 树形结构展示、搜索功能、详情查看按钮 |
| 经营主体管理 | 主体列表 | 数据表格、搜索功能、筛选功能、添加/编辑/删除按钮 |
| 经营主体管理 | 主体详情页 | 标签页展示不同信息模块、表单字段、附件上传、操作按钮 |
| 生产管理 | 生产记录明细 | 数据表格、搜索功能、筛选功能、详情查看按钮 |
| 统计分析 | 统计看板 | 图表展示（柱状图、折线图、饼图等）、数据卡片、时间筛选 |
| 系统管理 | 用户管理 | 数据表格、搜索功能、筛选功能、添加/编辑/删除按钮、密码重置按钮 |
| 系统管理 | 角色管理 | 数据表格、搜索功能、添加/编辑/删除按钮、权限分配按钮 |
| 系统管理 | 权限配置 | 树形结构展示、权限勾选、保存按钮 |

### 4.3 自适应
- **设计理念**：采用响应式设计，确保在不同屏幕尺寸下都能正常显示
- **桌面端**：优先考虑1920x1080分辨率，支持1280x720及以上分辨率
- **平板端**：支持768px-1024px宽度的平板设备
- **移动端**：支持320px-767px宽度的移动设备，但主要针对桌面端优化

## 5. 技术架构
### 5.1 技术栈
- **前端**：Vue 3 + Vben 2.0
- **UI框架**：Vben 2.0组件库
- **状态管理**：Pinia
- **路由**：Vue Router
- **HTTP客户端**：Axios
- **图表库**：ECharts
- **构建工具**：Vite

### 5.2 系统架构
- **单页应用**：采用Vue 3单页应用架构
- **组件化设计**：将页面拆分为可复用的组件
- **模块化管理**：按功能模块组织代码
- **响应式布局**：适配不同屏幕尺寸

## 7. 实现路径
1. **需求分析与规划**：分析用户需求，规划系统功能和页面结构。
2. **技术选型与架构设计**：选择合适的技术栈，设计系统架构。
3. **前端开发**：基于Vue 3和Vben 2.0开发前端页面和组件。
4. **数据结构设计**：设计系统所需的数据结构和数据库表结构。
5. **功能实现**：实现系统的各项功能，包括行政区划管理、经营主体管理、生产管理等。
6. **测试与优化**：进行系统测试，优化用户体验和性能。
7. **部署与上线**：部署系统到生产环境，确保系统正常运行。

## 8. 验收标准
1. **功能完整性**：系统实现所有需求文档中描述的功能。
2. **用户体验**：界面美观、操作简单、响应迅速。
3. **性能**：系统运行流畅，数据加载快速。
4. **兼容性**：在主流浏览器中正常运行。
5. **安全性**：系统具有基本的安全防护措施。

## 9. 风险评估
1. **数据安全**：需要确保系统数据的安全性，防止未授权访问和数据泄露。
2. **系统性能**：随着数据量的增加，需要确保系统性能不受影响。
3. **用户培训**：需要对用户进行培训，确保用户能够正确使用系统。
4. **技术依赖**：系统依赖于Vue 3和Vben 2.0等技术，需要关注这些技术的更新和维护。
